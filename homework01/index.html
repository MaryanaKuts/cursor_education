Перелік основних команд

  • git config --global user.name blabla - вказівка на імя користувача
  • git config --global user.email blabal@bla - вказівка на електронну адресу користувача
  • git init - створювати порожнє сховище Git або повторно ініціалізувати існуюче
  • git status - вказувати поточний статус Git репозиторію (робочого дерева)
  • git add - додавати вміст файлу до індексу перед комітом
  • git commit - створювати комміт, який є знімком змін до репозиторію
  • git log - переглядати історії змін
  • git checkout - перемикати гілки, здійснювати перехід в певну версію проекту по назві гілки, отримувати старі коміти і попередні версії файлів
  • git branch - створювати або видаляти гілки
  • git push - переносити локальну гілку в інший репозиторій, відправляти серію комітів
  • git pull - отримувати гілку з іншого репозиторію або локальної гілки та інтегрувати з ним
  • git merge - об’єднувати дві або більше історій розвитку разом
  • git clone - використовувати для створення копії певного сховища або гілки в сховищі

.gitignore файл - файл, що міститься в корені сховища і зберігає проігноровані файли. Файли .gitignore містять шаблони, які зіставляються з іменами файлів у вашому сховищі, щоб визначити, чи слід їх ігнорувати.

Advanced

  • git revert - cкасувати деякі наявні коміти
  • git rebase - перемістити гілки і уникнути непотрібних об"єнань комітів
  • git cherry-pick - застосувати зміни, внесені деякими наявними комітами
  • git stash - дозволяти тимчасово сховати зміни в робочій копії, щоб можна було застосувати їх пізніше
